# 四川麻将在线对战网站 · 产品需求文档（PRD）

- 版本：v1.0
- 日期：2026-06-16
- 状态：已评审

---

## 1. 产品概述

### 1.1 产品定位
一个在浏览器中运行的**四川麻将（血战到底）**在线对战网站。第一阶段为**单人对战 3 个 AI 机器人**，完整还原四川麻将血战到底的规则与记分；架构上预留向**真人房间制在线对战**演进的能力。

### 1.2 目标
- 完整实现四川麻将「血战到底」玩法：定缺、碰、杠、胡、查大叫、退税。
- 完整的翻番制记分系统，含高番型（龙七对系列、十八罗汉、金钩钓等）与特殊结算（杠上开花、海底捞月、门清）。
- 单桌多局记分：统计每人单局得分与累计总分。
- 规则型基础 AI，行为合理，可独立替换/升级（未来可接大模型）。
- 清晰可用的牌桌 UI（CSS 纯绘制牌面，无需美术素材）。

### 1.3 非目标（本阶段不做）
- 真人在线对战、房间匹配、WebSocket 联机（第二阶段，仅预留设计）。
- 账号注册登录、服务器持久化战绩、排行榜。
- 血流成河玩法（明确只做血战到底）。
- 查花猪重罚（明确不做，花猪仅按未听牌处理）。
- 动画、音效、精美牌面素材（后续增强项）。

---

## 2. 用户与场景

### 2.1 目标用户
熟悉四川麻将、想在线练习或消遣的单机玩家。

### 2.2 核心场景
1. 玩家进入开房页，配置底分、封顶、局数等规则，开始游戏。
2. 玩家在牌桌上与 3 个 AI 进行血战到底对局：定缺 → 摸打 → 碰杠胡。
3. 每局结束查看结算明细（番型、杠分、查大叫、退税）。
4. 多局打完查看累计总分与最终排名。

## 3. 功能需求

### 3.1 开房与规则配置
| 配置项 | 选项 | 默认 |
|---|---|---|
| 底分 | 1 / 2 / 5 | 1 |
| 封顶番数 | 5 / 7 / 10 / 不封顶 | 7 |
| 总局数 | 可设整数（如 4 / 8 / 16） | 8 |
| 抢杠胡 | 开 / 关 | 开 |
| 退税 | 开 / 关 | 开 |

### 3.2 牌与牌堆
- 仅万（W）、条（T）、筒（B）三门，每门 1–9，每张 4 枚，共 **108 张**。
- 洗牌发牌：每家起手 13 张，庄家 14 张（庄家先打）。

### 3.3 定缺
- 开局每家必须从三门花色中选一门「定缺」。
- AI 自动定缺（选张数最少的一门）。
- **未打完缺门牌不能胡**：胡牌时手中不得含定缺花色的牌。

### 3.4 摸牌与出牌
- 按座位顺序轮转：摸一张 → 打一张。
- 玩家点击手牌出牌；AI 自动决策（带延迟模拟思考）。
- 缺门牌在手牌中高亮提示，引导优先打出。

### 3.5 碰 / 杠 / 胡 / 过
- **碰**：别家打出与手中对子相同的牌，可碰（亮明 3 张）。
- **杠**：
  - 明杠（直杠）：别家打出第 4 张，杠。
  - 暗杠：自己手中摸齐 4 张，杠。
  - 补杠（巴杠/转弯杠）：已碰的牌再摸到第 4 张，杠。
- **胡**：满足胡牌牌型且已打完缺。点炮胡 / 自摸胡。
- **过**：放弃当前可执行的碰/杠/胡。
- 操作按钮**仅在引擎判定合法时亮起**。

### 3.6 血战到底流程（核心）
- 某家胡牌后**立即下桌**：停止摸打，本局不再参与，**单局每家最多胡一次**。
- 本局在剩余未胡玩家间继续，直到：
  - **3 家已胡**（只剩 1 家未胡），或
  - **牌墙摸完**（流局）。
- 抢杠胡：补杠时若别家可用该张胡牌，可抢杠胡（+1 番），补杠失败。

### 3.7 记分系统

#### 3.7.1 翻番制
- `分数 = 底分 × 2^番数`，番数受封顶限制。
- 番数 = 牌型基础番（取最高一种）+ 加番项。

#### 3.7.2 牌型基础番（互斥取最高）
| 番型 | 番数 |
|---|---|
| 平胡 | 0 |
| 对对胡 | 1 |
| 金钩钓（对对胡全副露单吊） | 2 |
| 清一色 | 2 |
| 七对 | 2 |
| 清对（清一色对对胡） | 3 |
| 清金钩钓 | 4 |
| 龙七对（七对含 1 组 4 张） | 3 |
| 清七对 | 4 |
| 双龙七对（含 2 组 4 张） | 4 |
| 十八罗汉（四杠 + 将） | 5 |
| 三龙七对（含 3 组 4 张） | 5 |
| 清龙七对 | 5 |
| 清双龙七对 | 6 |
| 清十八罗汉 | 7 |
| 清三龙七对 | 7 |

#### 3.7.3 加番项（每项 +1 番，可叠加）
| 加番 | 规则 |
|---|---|
| 根 | 任意 4 张相同（杠/刻子+暗藏第4张）每组 +1 番；龙七对系列的 4 张已体现为番型，**不再计根**；十八罗汉的 4 个杠**额外计 4 根（+4 番）** |
| 自摸 | 自己摸到胡牌 +1 番 |
| 杠上开花 | 开杠补摸的牌自摸胡 +1 番 |
| 海底捞月 | 摸牌墙最后一张自摸胡 +1 番；海底炮 = 别人打出最后一张点你胡 |
| 门清 | 全程无碰、无明杠（**暗杠不破门清**）+1 番，可与任意牌型叠加 |
| 抢杠胡 | 抢补杠成功 +1 番 |

#### 3.7.4 付分规则
- **点炮**：仅放炮那家付番型分（一对一）。
- **自摸**：其余三家各付番型分（含 +1 番自摸）。

#### 3.7.5 杠分（刮风下雨，当场结算）
| 杠型 | 收分 | 付方 |
|---|---|---|
| 明杠（直杠） | 2 × 底分 | 放杠那家单独付 |
| 暗杠 | 每家 2 × 底分（共 6） | 其余 3 家各付 |
| 补杠（巴杠） | 每家 1 × 底分（共 3） | 其余 3 家各付 |

### 3.8 流局结算
牌墙摸完仍有人未胡时：
- **查大叫（只做查大叫，不做查花猪）**：没听牌的玩家赔给**听牌但未胡**的玩家，赔付额 = 各听牌玩家「听的最大番型分」。已胡下桌的玩家不参与流局补偿。花猪（仍含三门）仅按未听牌处理，不重复罚。
- **退税**：只有**既没胡、又没听**的玩家，退还本局已收的杠分。听牌未胡、已胡下桌的玩家**不退**。

### 3.9 多局与记分板
- 常驻记分板显示每家累计总分 + 当前局数（如 3/8）。
- 单局结束弹结算面板：每家本局番型/杠分/查大叫/退税明细 + 本局净得分。
- 全部局数打完弹总结算：累计总分排名。
- 数据存于前端内存/会话，刷新即重置（本阶段不持久化）。

### 3.10 AI 机器人
- 规则型基础 AI：缺门优先打 → 向听数最小化 → 孤张优先 → 能胡就胡 + 简单碰杠启发式。
- 定缺：选张数最少的一门。
- 决策走引擎合法性校验，保证与真人同一裁决。
- 策略模块化（`BotStrategy` 接口），未来可接大模型（`LLMBot`）。

---

## 4. 非功能需求
- **正确性**：记分零和守恒（任意时刻四家分数总和恒为 0），作为记分正确性总闸。
- **可移植性**：游戏引擎为纯逻辑模块，不依赖 DOM/网络，第二阶段可平移至服务端。
- **性能**：单机本地运算，胡牌/向听判定毫秒级。
- **可测性**：引擎纯函数，支持固定牌堆注入做确定性测试。

---

## 5. 验收标准
1. 可配置开房并开始一局完整血战到底对局。
2. 定缺、碰、杠、胡、过交互正确，非法操作不可执行。
3. 所有番型与加番项记分正确（有测试覆盖）。
4. 刮风下雨杠分、抢杠胡、查大叫、退税结算正确。
5. 血战到底流程正确：胡后下桌、单局只胡一次、3 家胡或流局结束。
6. 多局累计记分与排名正确，四家分数零和守恒。
7. 引擎单元测试覆盖率 ≥ 90%，AI 四家互打冒烟测试稳定通过。
